24.7 Event codes

Accessor macro: int MARPA_EVENT_NONE

Applications should never see this event. Event value: Unspecified. Suggested message: "No event".

Accessor macro: int MARPA_EVENT_COUNTED_NULLABLE

A nullable symbol was used as either the separator for, or the right hand side of, a sequence. This is not allowed.

This event is generated by the marpa_g_precompute() method. See marpa_g_precompute(). Event value: The ID of the symbol. Suggested message: "This symbol is a counted nullable".

Accessor macro: int MARPA_EVENT_EARLEY_ITEM_THRESHOLD

The MARPA_EVENT_EARLEY_ITEM_THRESHOLD event indicates that an application-settable threshold on the number of Earley items has been reached or exceeded. See marpa_r_earley_item_warning_threshold_set().

This event is generated by the marpa_r_earleme_complete() method. See marpa_r_earleme_complete(). Event value: The current Earley item count. Suggested message: "Too many Earley items".

Accessor macro: int MARPA_EVENT_EXHAUSTED

This event occurs when the parse is exhausted.

This event is generated by the marpa_r_earleme_complete() (see marpa_r_earleme_complete) and marpa_r_start_input() (see marpa_r_start_input) methods. Event value: Unspecified. Suggested message: "Recognizer is exhausted".

Accessor macro: int MARPA_EVENT_LOOP_RULES

One or more rules are loop rules — rules that are part of a cycle. Parsing with a grammar than contains a loop rule is deprecated. See Cycles.

This event is generated by the marpa_g_precompute() method. See marpa_g_precompute(). Event value: The count of loop rules. Suggested message: "Grammar contains a infinite loop".

Accessor macro: int MARPA_EVENT_NULLING_TERMINAL

The MARPA_EVENT_NULLING_TERMINAL event occurs only if the LHS terminals feature is in use. The LHS terminals feature is deprecated. See LHS terminals.

This event is generated by the marpa_g_precompute() method. See marpa_g_precompute(). Event value: The ID of the symbol. Suggested message: "This symbol is a nulling terminal".

Accessor macro: int MARPA_EVENT_SYMBOL_COMPLETED

The recognizer can be set to generate an event when a symbol is completed. (A symbol is “completed” iff any rule with that symbol as its LHS is completed.) To set a completed symbol event, use the marpa_g_symbol_is_completion_event_set() method. See marpa_g_symbol_is_completion_event_set()

This event is generated by the marpa_r_earleme_complete() method. See marpa_r_earleme_complete(). Event value: The ID of the completed symbol. Suggested message: "Completed symbol".

Accessor macro: int MARPA_EVENT_SYMBOL_EXPECTED

The recognizer can be set to generate an event when a symbol is expected as a terminal. To set an expected terminal event, use the marpa_r_expected_symbol_event_set() method. See marpa_r_expected_symbol_event_set(). Note that this event only triggers if the symbol is expected as a terminal. Predicted symbols that are not expected as terminals do not trigger this event.

This event is generated by the marpa_r_earleme_complete() (see marpa_r_earleme_complete) and marpa_r_start_input() (see marpa_r_start_input) methods. Event value: The ID of the expected symbol. Suggested message: "Expecting symbol".

Accessor macro: int MARPA_EVENT_SYMBOL_NULLED

The recognizer can be set to generate an event when a symbol is nulled – that is, recognized as a zero-length symbol. To set an nulled symbol event, use the recognizer’s marpa_g_symbol_is_nulled_event_set() method. See marpa_g_symbol_is_nulled_event_set().

This event is generated by the marpa_r_earleme_complete() (see marpa_r_earleme_complete) and marpa_r_start_input() (see marpa_r_start_input) methods. Event value: The ID of the nulled symbol. Suggested message: "Symbol was nulled".

Accessor macro: int MARPA_EVENT_SYMBOL_PREDICTED

The recognizer can be set to generate an event when a symbol is predicted. To set an predicted symbol event, use the recognizer’s marpa_g_symbol_is_prediction_event_set() method. See marpa_g_symbol_is_prediction_event_set(). Unlike the MARPA_EVENT_SYMBOL_EXPECTED event, the MARPA_EVENT_SYMBOL_PREDICTED event triggers for predictions of both non-terminals and terminals.

This event is generated by the marpa_r_earleme_complete() (see marpa_r_earleme_complete) and marpa_r_start_input() (see marpa_r_start_input) methods. Event value: The ID of the predicted symbol. Suggested message: "Symbol was predicted".
